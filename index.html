Why TOC and headings with anchors and numbers? ðŸ¤”

  • TOC gives a bird's eye view of the article. It allows the readers to jump to a particular section they are interested in from there.
  • Numbers before headings make the hierarchical position of a section clear.
  • Clicking on an anchor makes it super easy to both jump to that section and get its link in the URL so we can easily refer to it from elsewhere if needed.

However, as of July 2022, there is no automatic way to do these things in the freeCodeCamp blog editor. So I've made this to help freeCodeCamp authors generate beautiful TOC and headings with inlined CSS so they can copy and paste it into HTML cards to replicate it instantly.

Note: For styling it uses inline CSS styles because attaching external stylesheets/scripts is not approved by the style guide.
